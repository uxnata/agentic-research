# Агентная система количественного исследования

Автономный AI-исследователь, который проводит полный цикл количественного анализа данных: от аудита качества до практических рекомендаций.

## Что это

LLM (Claude / GPT-4o / Gemini через OpenRouter) выступает в роли исследователя. Он получает набор статистических инструментов и самостоятельно решает на каждом шаге: какой тест запустить, нужна ли проверка допущений, стоит ли копать подгруппы, формулировать новую гипотезу или остановиться.

Это не линейный скрипт с захардкоженными гипотезами. Это agentic loop с tool use, где стратегия анализа адаптируется к данным.

## Пайплайн

```
Фаза 0  DATA QUALITY AUDIT     пропуски, выбросы, мультиколлинеарность, power analysis
Фаза 1  EDA                    описательные, корреляции, распределения, графики
Фаза 2  HYPOTHESES             генерация гипотез из находок EDA
Фаза 3  TESTING                стат. тесты с автоматическим выбором метода
Фаза 4  REFLECTION             неожиданные результаты -> новые гипотезы -> углубление
Фаза 5  CAUSAL REASONING       partial correlations, медиация, поиск конфаундов
Фаза 6  ROBUSTNESS CHECKS      bootstrap CI, outlier sensitivity, split-half
Фаза 7  RECOMMENDATIONS        сегментация по группам риска, приоритезированные рекомендации
Фаза 8  SYNTHESIS              финальный отчёт в Markdown
```

## Инструменты агента (15 штук)

| Инструмент | Назначение |
|---|---|
| `describe_dataset` | Обзор датасета: shape, типы, статистики |
| `audit_data_quality` | Аудит качества, возвращает GO / CAUTION / STOP |
| `get_correlations` | Топ-N корреляций |
| `get_distribution` | Распределение переменной + тест нормальности |
| `compare_groups` | Автовыбор t-test / ANOVA / Mann-Whitney / Kruskal-Wallis |
| `test_correlation` | Pearson + Spearman с интерпретацией эффекта |
| `test_association` | Хи-квадрат + V Крамера |
| `run_regression` | Множественная регрессия с VIF и кросс-валидацией |
| `create_plot` | scatter / box / hist / heatmap / violin / bar |
| `partial_correlation` | Частная корреляция для поиска конфаундов |
| `mediation_analysis` | Прямой / косвенный эффект, процент медиации |
| `robustness_check` | Bootstrap, outliers, split-half, сравнение методов |
| `segment_analysis` | Группы риска по целевой переменной |
| `check_subgroups` | Модерация: связь в подгруппах |
| `get_research_state` | Текущее состояние: гипотезы, тесты, что проверено |

## Синтетический датасет

`generate_dataset.py` создаёт датасет из 3000 сотрудников IT-компании (38 переменных) с заложенными закономерностями:

- experience -> salary (линейная, r ~ 0.7)
- sleep -> productivity (линейная, r ~ 0.35)
- work_format -> job_satisfaction (удалёнка > гибрид > офис)
- overtime -> burnout -> turnover_intention -> left_company (цепочка)
- coffee -> productivity (нелинейная, inverted U, пик на 3 чашках)
- sleep * overtime -> productivity (модерация: при переработках сон важнее)

## Архитектурные решения

**Почему агент, а не скрипт.** Исследование — не конвейер. Обнаружив неожиданный результат, исследователь меняет план. Агент воспроизводит это: рефлексия может породить новые гипотезы и запустить дополнительные тесты.

**Почему не LangChain / n8n.** Tool use уже встроен в API. Для agentic loop достаточно 50 строк. Фреймворки добавляют абстракции без пользы для этой задачи.

**Почему высокоуровневые tools, а не exec().** Агент формулирует вопрос к данным (`compare_groups(dv="burnout", groupby="role")`), а не пишет код. Инструмент сам проверяет допущения и выбирает тест. Это надёжнее и воспроизводимее.

**Почему один агент с фазами, а не цепочка.** Единый контекст, меньше токенов, предсказуемое поведение. Фаза — это смена промпта, а не новая сущность.

## Ограничения

- Синтетические данные с заранее заложенными закономерностями
- Кросс-секционный дизайн (нет причинности даже с mediation)
- Нет SEM (только Baron & Kenny mediation)
- Нет ML-предикторов (только линейная регрессия)
- Нет FDR-коррекции (только фиксированный alpha)
- Нестабильность: повторный запуск даст другие гипотезы
